2. Introduction
The Hospital Sync Management System is a web-based application developed using the Django framework. The purpose of the system is to manage hospital operations such as patient records, appointments, and administrative tasks in a centralized and efficient manner.
Traditional hospital record systems are often manual or fragmented, leading to delays, errors, and inefficiencies. This system aims to improve data management by providing a digital solution that allows secure and structured access to hospital information.
________________________________________
3. Problem Statement
Hospitals often face challenges such as:
•	Inefficient patient record management 
•	Data duplication and loss 
•	Difficulty in tracking appointments and medical history 
•	Lack of centralized system for staff and admin use 
This project addresses these issues by developing a centralized web application for managing hospital data efficiently.

6. System Design
The system follows the MVC (Model-View-Controller) architecture used by Django:
•	Models: Define database structure (patients, appointments, etc.) 
•	Views: Handle logic and processing 
•	Templates: Display data to users 
Architecture Flow:
User → View (Django) → Model (Database) → Response → Template
